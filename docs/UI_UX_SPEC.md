\# UI/UX Specification — Nuzlocke Companion



\## Main screen layout



The main screen should look like an interactive Nuzlocke layout.



Structure:



\- Top bar:

&#x20; - Game name

&#x20; - Challenge type

&#x20; - Settings/edit mode buttons



\- Main gameplay area:

&#x20; - Large central/left area.

&#x20; - Placeholder for gameplay in MVP.

&#x20; - Should visually feel like the game is the focus.



\- Right team panel:

&#x20; - 6 Pokémon slots.

&#x20; - Each slot shows:

&#x20;   - sprite/avatar

&#x20;   - nickname

&#x20;   - optional species

&#x20;   - optional level



\- Bottom status bar:

&#x20; - Life counter: one heart icon plus number, for example ❤️ 4.

&#x20; - Badges: 8 icons, grey when not obtained, colored when obtained.

&#x20; - Level cap: example “Level Cap: 21”.

&#x20; - Current route: example “Route 24”.

&#x20; - Capture status: Available / Used / Failed / Not applicable.



\## Visual direction

The first layout should be inspired by Pokémon video layouts:

\- Big gameplay frame.

\- Right-side team list.

\- Bottom status section.

\- Blue or purple theme.

\- Clean borders.

\- Rounded but game-like panels.

\- Minimal clutter.



\## Interaction rules

\- Player can manually edit team.

\- Player can increase/decrease lives.

\- Player can toggle badges.

\- Player can change current route.

\- Player can toggle capture status.

\- No complex logs or database UI in MVP.
